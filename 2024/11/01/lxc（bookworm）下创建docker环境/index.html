这边就不科普lxc是啥，dockers是啥了，直接开整！

说在前面

首先在创建docker环境前，可以考虑在lxc内安装一些服务，比如samba等，方便后期上传下载，这边略过，教程一大把。

安装docker环境

其实非常简单，一条命令即可，如下：
1. apt install docker.io -y

安装docker-compose环境


找到docker-compose下载链接，在lxc控制台内输入：
1. wget -c https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-linux-x86_64 -O /usr/local/bin/docker-compose && chmod +x docker-compose


可以通过docker info和docker-compose –version命令查看安装正常与否。

举2个docker实例

简单docker部署，可以分为docker cli和docker-compose部署，docker cli的话以一个非常火的项目memos来举例，非常简单，一条命令如下：
1. docker run -itd --name memos --restart unless-stopped -p 5230:5230 -v /opt/appdata/memos:/var/opt/memos neosmemo/memos
等系统拉取镜像成功并实例化容器，显示如下：

回浏览器，开启新页面，输入lxc的ip地址加端口5230回车，可以看到项目部署成功。

docker-compose是用来部署那些相对较复杂的容器组合，举例如下：
在/root目录下有一个notea.yml的文件，使用cat命令显示为：

在控制台输入：
1. docker-compose -f notea.yml up -d

等系统拉取镜像成功并实例化容器。
回浏览器，开启新页面，输入lxc的ip地址加端口13320回车，可以看到项目部署成功。

本章不介绍2个容器的使用方法，仅用来做展示说明。
